# D-Lite Chat Service

This service provides real-time chat for the D-Lite system using Node.js, Express, Socket.IO, and Supabase PostgreSQL. It supports one-to-one chats and group chats by using Socket.IO rooms.

## Features

- Runs on port `4002`
- Real-time messaging with Socket.IO
- Typing indicators
- Online and offline user status updates
- Message storage in Supabase PostgreSQL
- REST endpoint for loading chat history

## Message Schema

The service expects a `messages` table in Supabase with these columns:

- `id`
- `chat_id`
- `sender_id`
- `content`
- `type`
- `created_at`

Example SQL:

```sql
create table if not exists public.messages (
  id bigint generated by default as identity primary key,
  chat_id text not null,
  sender_id text not null,
  content text not null,
  type text not null default 'text',
  created_at timestamptz not null default now()
);
```

## Socket Events

### Client to server

- `join_chat`
- `send_message`
- `typing`
- `stop_typing`

### Server to client

- `receive_message`
- `typing`
- `stop_typing`
- `user_status`
- `socket_error`

## How Rooms Work

- For a 1-to-1 chat, both users join the same room like `chat:direct-12-15`
- For a group chat, all members join the same room like `chat:group-42`

The service does not decide room membership itself. The client joins a room by sending:

```json
{
  "chatId": "group-42"
}
```

with the `join_chat` event.

## REST API

### `GET /messages/:chatId`

Returns all messages for a chat ordered by `created_at`.

## Environment Variables

Create a `.env` file inside `D-Lite-chat-service`:

```env
PORT=4002
CORS_ORIGINS=http://localhost:4000,http://localhost:3000,http://localhost:5173
SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-supabase-service-role-key
```

You can use `SUPABASE_ANON_KEY` instead, but the service role key is usually better for backend inserts and reads.

## Example Client Flow

1. Connect to Socket.IO with `userId` in the handshake.
2. Emit `join_chat` with a `chatId`.
3. Emit `send_message` with `chatId`, `senderId`, `content`, and optional `type`.
4. Everyone in that room receives `receive_message`.
5. Emit `typing` and `stop_typing` for typing indicators.
